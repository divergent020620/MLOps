# Cube Studio 二开流程报告

## 一、问题分析

官方部署使用预构建镜像：
- 后端镜像：`ccr.ccs.tencentyun.com/cube-studio/kubeflow-dashboard`
- 前端镜像：`ccr.ccs.tencentyun.com/cube-studio/kubeflow-dashboard-frontend`

二开需要从源码构建自定义镜像并替换部署配置。

## 二、本地开发调试

### 1. 后端开发
```bash
cd install/docker
docker-compose up -d
docker exec -it docker-myapp-1 bash

# 容器内修改代码后
python myapp/run.py  # 热重载调试
```

### 2. 前端开发
```bash
cd myapp/frontend
npm install
npm run start  # 访问 http://localhost:3000
```

前端通过 `setupProxy.js` 代理到后端，支持热更新。

### 3. 关于角色管理
前端角色管理**未被注释**，位于 `routerConfig.tsx` 第 42-48 行：
```typescript
{
    path: '/security/roleList',
    title: '角色列表',
    menu_type: 'iframe',
    element: lazy2Compont(() => import("./pages/IframeTemplate"), 
             { url: '/roles/list/?_flt_2_name=' })
}
```

采用 iframe 嵌入 Flask-AppBuilder 原生管理页面，后端逻辑在 `flask_appbuilder` 框架中。

## 三、构建自定义镜像

### 1. 后端镜像构建

**首次或依赖变更时构建基础镜像**：
```bash
docker build --network=host \
  -t your-registry.com/cube-studio/kubeflow-dashboard:base-python3.9 \
  -f install/docker/Dockerfile-base .
  
docker push your-registry.com/cube-studio/kubeflow-dashboard:base-python3.9
```

**构建业务镜像**（包含代码改动）：
```bash
docker build --network=host \
  -t your-registry.com/cube-studio/kubeflow-dashboard:v1.0.0-custom \
  -f install/docker/Dockerfile .
  
docker push your-registry.com/cube-studio/kubeflow-dashboard:v1.0.0-custom
```

### 2. 前端镜像构建

**先本地构建前端产物**：
```bash
cd myapp/frontend
npm run build  # 生成 build/ 目录
```

**构建前端镜像**：
```bash
cd ../..  # 回到项目根目录
docker build --network=host \
  -t your-registry.com/cube-studio/kubeflow-dashboard-frontend:v1.0.0-custom \
  -f install/docker/dockerFrontend/Dockerfile .
  
docker push your-registry.com/cube-studio/kubeflow-dashboard-frontend:v1.0.0-custom
```

## 四、修改 K8s 部署配置

### 1. 修改后端部署
编辑 `install/kubernetes/cube/base/deploy-backend.yaml`：
```yaml
containers:
  - name: kubeflow-dashboard
    image: your-registry.com/cube-studio/kubeflow-dashboard:v1.0.0-custom
    imagePullPolicy: Always
```

### 2. 修改前端部署
编辑 `install/kubernetes/cube/base/deploy-frontend.yaml`：
```yaml
containers:
  - name: kubeflow-dashboard-frontend
    image: your-registry.com/cube-studio/kubeflow-dashboard-frontend:v1.0.0-custom
    imagePullPolicy: Always
```

### 3. 修改其他组件（如需要）
- `deploy-worker.yaml`：Celery 异步任务 worker
- `deploy-schedule.yaml`：Celery beat 定时任务
- `deploy-watch.yaml`：K8s 资源监控

这些组件也使用后端镜像，需同步修改。

## 五、部署更新

```bash

kubectl apply -f install/kubernetes/cube/base/deploy-frontend.yaml

```

## 六、版本管理建议

1. **镜像版本命名规范**：`v{major}.{minor}.{patch}-{feature}`
   - 例如：`v1.0.0-custom`、`v1.0.1-role-mgmt`

2. **Git 分支管理**：
   - `main`：跟随官方主分支
   - `dev-custom`：你的二开分支
   - 定期 rebase 官方更新

3. **配置管理**：
   - 使用 Kustomize overlays 管理不同环境配置
   - 敏感配置通过 ConfigMap/Secret 注入

## 七、常见问题

1. **Windows 开发注意事项**：
   - 设置 `git config --global core.autocrlf false`
   - shell 脚本使用 LF 换行符

2. **前端构建失败**：
   - 修改 `myapp/visionPlus/.eslintrc`，注释 unix linebreak 规则

3. **镜像构建加速**：
   - 使用国内镜像源（阿里云、腾讯云）
   - 配置 Docker 镜像加速器

4. **热更新不生效**：
   - docker-compose 环境不支持前端热更新
   - 使用 `npm run start` 本地开发

## 八、角色管理二开示例

如需自定义角色管理页面（替换 iframe 方式）：

1. **后端**：继承 `flask_appbuilder.security.views.RoleModelView`
2. **前端**：创建新的 React 页面替换 iframe
3. **路由**：修改 `routerConfig.tsx` 中的 `/security/roleList` 配置

参考现有的用户中心页面 `UserCenter.tsx` 实现模式。
