# Cube Studio 开源版 vs 企业版功能对比（完整修正版）

## 企业版功能屏蔽机制

### 1. 前端菜单屏蔽
通过 `myapp/views/home.py` 中的 `"disable": True` 或 `"hidden": 1` 标记

### 2. 镜像屏蔽
企业版镜像使用 `enterprise-` 前缀，标注为 `（todo）`，无法 pull 下载
- `ccr.ccs.tencentyun.com/cube-studio/notebook:enterprise-jupyter-ubuntu-cpu-pro`
- `ccr.ccs.tencentyun.com/cube-studio/notebook:enterprise-matlab-ubuntu-deeplearning`
- `ccr.ccs.tencentyun.com/cube-studio/notebook:enterprise-rstudio-ubuntu-bigdata`

### 3. Job Template 屏蔽
在 `myapp/init/init-job-template.json` 中标注 `(企业版)`

## 九个问题的完整修正答案

### 1. Hadoop 对接案例和功能 ✅

**开源版可用**：
- Bigdata Notebook 镜像（Hadoop 3.2.2 + Spark 3.1.3 + Hive 3.1.2 + Flink 1.15.1）
- Hadoop Job Template
- 配置文件和示例代码

**结论**：✅ 开源版完全支持

---

### 2. 资源隔离和权限控制 ⚠️

#### 资源隔离级别

**✅ 项目组级别（Project）**：开源版支持
- 独立的 namespace、node_selector、volume_mount
- 项目组内可配置资源组（org）

**✅ 用户级别**：开源版支持
- 用户个人目录（/mnt/$username）
- Project_User 表的 quota 和 org 字段

#### 权限分级

**后端权限体系**：✅ 开源版完整支持
- Flask-AppBuilder RBAC 系统
- 角色（Admin、Gamma、Public）
- 权限（can_list, can_add, can_edit, can_delete, can_show）
- 资源级权限控制（check_edit_permission）

**前端权限管理界面**：⚠️ 默认被注释
- 代码存在但被注释：`routerConfig.tsx` 第 135-137 行
- 包含：用户列表、角色列表、权限列表、权限视图关系
- **需要手动启用**：取消注释即可使用

**项目组内用户间隔离**：❌ 开源版不支持
- 项目组成员默认可以看到所有资源
- **需要二次开发**：添加用户级过滤器

**精准功能权限控制**：⚠️ 部分支持
- 后端支持，但前端界面默认关闭
- **需要二次开发**：启用前端界面并调整权限判断逻辑

## 企业版镜像 Pull 失败的原因

根据 issue 反馈和代码分析：
- 企业版镜像使用 `enterprise-` 前缀
- 这些镜像**不在公开镜像仓库中**
- 配置文件中标注为 `（todo）`
- **只有购买企业版才能获得访问权限**

示例：
```python
# 开源版可用
'ccr.ccs.tencentyun.com/cube-studio/notebook:jupyter-ubuntu-deeplearning'

# 企业版不可用（pull 失败）
'ccr.ccs.tencentyun.com/cube-studio/notebook:enterprise-jupyter-ubuntu-cpu-pro'
```
